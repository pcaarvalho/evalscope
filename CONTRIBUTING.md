# Contributing

## Operating standard

- Keep scope narrow.
- Prefer simple code over clever abstractions.
- Write the smallest useful release first.
- Document decisions that change the product direction.

## Pull requests

- Explain the user-facing change.
- State what was tested.
- Call out known gaps or follow-up work.

## Releases

- Ship small, credible milestones.
- Do not cut a release that damages trust.
